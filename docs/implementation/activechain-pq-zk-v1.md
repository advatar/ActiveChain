# ActiveChain PQ-ZK v1 implementation and assurance record

This document is the reproducible evidence index for issue #31. The implementation is a pinned
RISC Zero zk-STARK guest and host verifier, not a new cryptographic primitive designed by the
ActiveChain team.

## Security posture

- Transparent proof path: no trusted setup and no elliptic-curve verifier dependency in the
  conforming receipt path.
- Witness privacy: private inputs are supplied to the zkVM and only the versioned journal is public.
- Post-quantum posture: conditional on the hash-based STARK/FRI assumptions of the pinned engine.
- Development status: formally checked application invariants; third-party audit pending.

RISC Zero is pinned exactly because proof formats, control identifiers, and security parameters are
protocol inputs. The host enables `disable-dev-mode` so an environment variable cannot silently
replace a proof with a development receipt. Groth16 compression is excluded because it introduces
elliptic-curve and trusted-setup assumptions that do not satisfy this profile.

## What formal verification means here

The Lean model proves conservation, deterministic successor state, one-shot nullifier admission,
cooldown/penalty monotonicity, and atomic admission properties for the abstract billboard relation.
Those theorems do not prove that RISC Zero, Rust, the RISC-V compiler, SHA-3, Poseidon2, FRI, or this
machine are bug-free. Rust differential tests connect concrete fixtures to the reference billboard
verifier; this is evidence, not a full refinement proof.

## Reproduction

Run:

```sh
cargo test -p activechain-pq-zk --all-features --release
./scripts/check-formal-models.sh
cargo test --workspace --all-targets
cargo fmt --all -- --check
```

The default `reproducible-build` feature builds the guest in the RISC Zero 1.94.1 Docker environment
pinned by manifest digest, at the fixed `/src` path. Published image IDs and release artifacts MUST
come from that build. `--no-default-features` permits faster native guest builds for development,
but those builds embed host source paths and are not protocol-reproducible across checkout
locations.

Real proving requires the pinned RISC Zero 3.0.5 Rust toolchain and `r0vm`. Verification does not
require the witness. Benchmark output and deterministic public vectors are generated by the crate's
tests and checked into `testing/vectors/` once the relation is frozen.

On the 2026-07-22 Apple Silicon development host, the final ignored release security-gate test
generated, verified, malformed-journal-tested, and substitution-tested a real succinct receipt in
183.22 seconds after compilation. This single-host observation is reproducibility evidence, not a
performance guarantee.

## Audit target

An external review must cover P-111, dependency and feature pinning, guest image reproducibility,
the complete guest relation, journal encoding, image-ID and receipt-kind checks, the billboard
admission adapter, adversarial tests, formal models and their stated refinement boundary, and build
supply-chain controls. Until that report is public, the landing page must retain “third-party audit
pending.”
